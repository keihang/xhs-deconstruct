# 风控规避指南

小红书有严格的风控机制，需要小心操作。

## 核心原则

**慢是关键**：宁可慢一点，也不要触发风控。触发后要等很久才能恢复。

## 操作间隔

### 必须遵守的间隔
- **搜索操作**：每次间隔 **15秒**
- **读取帖子**：每次间隔 **15秒**
- **下载封面**：每次间隔 **15秒**
- **批量操作**：处理完一个博主，等 **5分钟** 再处理下一个

### 为什么是15秒？
- 小红书的风控阈值大约是10秒
- 15秒是安全边界，既不太慢也不触发风控
- 如果网络慢，可以增加到20秒

## 风控触发后的处理

### 轻度风控（返回空数据）
- **表现**：搜索结果为空，或返回空数组
- **处理**：等 **5-10分钟** 再试
- **预防**：降低操作频率

### 中度风控（返回"安全验证"）
- **表现**：读取帖子内容报错"安全验证"
- **处理**：等 **15-30分钟** 再试
- **预防**：严格遵守15秒间隔

### 重度风控（IP被封）
- **表现**：所有操作都失败，即使等很久也没用
- **处理**：等 **1-2小时** 再试，或换个网络环境
- **预防**：控制单次操作数量（不超过50次）

## 操作数量控制

### 单次操作限制
- **搜索**：每次最多20条（limit=20）
- **读取帖子**：单次最多10篇
- **下载封面**：单次最多10个

### 批量操作限制
- **单个博主**：最多处理15篇帖子
- **单次会话**：最多处理3个博主
- **总操作数**：单次会话不超过100次操作

### 为什么有这些限制？
- 操作次数越多，触发风控的概率越高
- 小红书会监控单IP的操作频率
- 控制数量是预防风控的最有效方法

## URL格式要求

### ✅ 正确的URL格式
```
https://www.xiaohongshu.com/explore/xxxxx?xsec_token=xxxxx
```
- 带 `xsec_token` 参数
- 从搜索结果里获取

### ❌ 错误的URL格式
```
https://www.xiaohongshu.com/explore/xxxxx
```
- 不带 `xsec_token` 参数
- 从其他地方获取（如浏览器地址栏）

### 为什么URL格式很重要？
- `xsec_token` 是小红书的验证令牌
- 没有令牌，请求会被拒绝
- 这是小红书的安全机制

## 下载注意事项

### 封面下载
- ✅ 用搜索结果URL下载
- ❌ 用explore URL下载会失败
- 下载后检查文件是否有效（有时会下载成mp4）

### 文件格式问题
- 正常情况：`.jpg` 或 `.png`
- 异常情况：`.mp4`（视频格式）
- 处理方法：删除异常文件，跳过这篇帖子

## 会话管理

### 单次会话建议
- 时间控制在 **30分钟以内**
- 操作次数控制在 **100次以内**
- 处理博主数量控制在 **3个以内**

### 多次会话建议
- 每次会话间隔 **1小时以上**
- 不要连续多次会话
- 观察是否有风控迹象

## 监控与预警

### 风控预警信号
- 响应变慢（从1秒变成3-5秒）
- 返回数据变少（从20条变成10条）
- 返回空数据（从有数据变成空数组）
- 返回错误信息（"安全验证"、"请求频繁"）

### 预警处理
- 立即停止操作
- 等待5-10分钟
- 如果恢复，继续操作（降低频率）
- 如果没恢复，等待更长时间（30分钟以上）

## 最佳实践

### 操作前
1. 检查网络环境（稳定、不频繁切换）
2. 准备好搜索关键词和筛选条件
3. 规划好操作数量（不要贪多）

### 操作中
1. 严格遵守15秒间隔
2. 监控响应速度和返回数据
3. 发现异常立即停止

### 操作后
1. 保存搜索结果（JSON文件）
2. 检查下载的文件是否有效
3. 记录操作数量和耗时

## 常见错误

### 错误1：操作太快
- **表现**：连续快速操作，没有间隔
- **后果**：触发风控，需要等待很久
- **正确做法**：每次操作间隔15秒

### 错误2：操作太多
- **表现**：单次会话处理太多博主或帖子
- **后果**：触发风控，需要等待很久
- **正确做法**：单次会话控制在100次操作以内

### 错误3：URL格式不对
- **表现**：用explore URL读取或下载
- **后果**：请求失败，返回空数据或错误
- **正确做法**：用搜索结果URL（带xsec_token）

### 错误4：不检查文件
- **表现**：下载后不检查文件是否有效
- **后果**：文件损坏或格式错误
- **正确做法**：下载后检查文件大小和格式

## 应急预案

### 触发轻度风控
1. 立即停止操作
2. 等待5-10分钟
3. 用小量操作测试（如搜索1条）
4. 如果恢复，继续操作（降低频率）

### 触发中度风控
1. 立即停止操作
2. 等待15-30分钟
3. 用小量操作测试
4. 如果没恢复，等待更长时间（1小时以上）

### 触发重度风控
1. 立即停止操作
2. 等待1-2小时
3. 如果还没恢复，换个网络环境
4. 或者改天再试

## 总结

**核心原则**：
1. 慢是关键（15秒间隔）
2. 控制数量（单次100次操作以内）
3. 监控异常（响应变慢、数据变少）
4. 及时停止（发现异常立即停止）

**记住**：宁可慢一点，也不要触发风控。触发后要等很久才能恢复，得不偿失。
